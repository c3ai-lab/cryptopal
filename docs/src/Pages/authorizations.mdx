# Payments API

Call the Payments API to authorize payments, capture authorized payments, refund payments that have already been captured, and show payment information. Use the Payments API in conjunction with the Orders API.

<h2>Authorizations</h2>

Use the /authorizations resource to show details for, capture payment for, reauthorize, and void authorized payments.

## Show details for authorized payment

<Route method="POST" path="payments/authorizations/{authorization_id}" />

Shows details for an authorized payment, by ID.

## Void authorized payment

<Route method="GET" path="payments/authorizations/{authorization_id}/void" />

Voids, or cancels, an authorized payment, by ID. You cannot void an authorized payment that has been fully captured.

### Header parameters

<RequestHeader />

### Path parameters

<ShowDetailsPath />

### Response

<div>
  A successful request returns the
  <span className="highlight-code">HTTP 204 No Content</span> status code with no
  JSON response body.
</div>
